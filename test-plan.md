# SauceDemo Manual QA - Test Plan

## 1. Introduction
This document outlines the test plan for manual Quality Assurance (QA) of the [SauceDemo](https://www.saucedemo.com/) web application. The purpose of this plan is to ensure the application meets all specified requirements and is free of critical bugs.

## 2. Objectives
- Validate all critical user flows: login, product selection, cart, checkout, and logout.
- Identify and report functional, UI, and usability issues.
- Ensure compatibility across browsers (Chrome, Firefox, Safari).

## 3. Scope
**In Scope:**
- Manual functional testing of the SauceDemo web app features.
- UI testing on desktop browsers.
- Regression testing for fixed bugs.

**Out of Scope:**
- Mobile browser and device testing.
- Performance and load testing.

## 4. Test Approach
- Positive and negative scenario-based manual testing.
- Use of real user credentials provided by SauceDemo.
- Step-by-step documentation of test cases and actual results.

## 5. Test Environment
- **URL:** https://www.saucedemo.com/
- **Browsers:** Chrome (latest), Firefox (latest), Safari (latest)
- **Test Data:** Standard users provided by SauceDemo

## 6. Test Cases Overview
- **Login Functionality**
- **Product Inventory Page**
- **Product Details**
- **Add to Cart/Remove from Cart**
- **Cart and Checkout**
- **Order Completion**
- **Logout**

## 7. Deliverables
- Test Plan document
- Test cases (to be added in a separate file)
- Bug/issue report (if any)
- Test summary report

## 8. Timeline
- Test Plan Preparation: 1 day
- Test Case Creation: 1 day
- Manual Execution: 1-2 days
- Reporting: 0.5 day

## 9. Risks & Mitigation
- **Unavailability of test environment** - Coordinate with dev team.
- **Test data changes** - Validate test data before each cycle.
